# 產品資料比對

## 目標

使用 `參考資料/產品資料輸出*.CSV` 檢查進貨商品試算表是否已建檔，將比對欄位寫入檢查用 xlsx。商品名稱候選比對預設由本地 RapidFuzz 執行，不交給 AI 逐筆判定；AI 只可用於整理低信心項目的回報、說明疑點或產生使用者確認清單。不得輸出 JSON；JSON 只可作為舊資料理解參考，不可作為本 Skill 的正式輸入或輸出。

## 必要檢查

產品資料 CSV 的檔案建立時間或修改時間必須至少一個是台北時區今天。建立時間與修改時間都不是今天時，停止並提醒使用者更新檔案，不得繼續比對或輸出正式檔案。檔名不需要包含日期。

## 工作流程

1. 接收使用者提供的 `建檔進貨用/進貨圖片轉試算表/*.xlsx`。從試算表自動尋找表頭列並讀取欄位：`產品代號`、`品名`、`數量`、`進價`、`金額`；若有 `零售價` 欄位也一併保留。
   - 讀取商品列時，若 `產品代號` 或 `品名` 欄出現 `總價格`、`總價`、`總計`、`合計`、`小計`，該列視為摘要列，不得列入商品比對或後續匯入。
2. 若輸入檔名已包含 `_產品比對檢查`，停止並提醒使用者改用原始進貨試算表，不要重複加工檢查檔。
3. 找到或使用使用者指定的產品資料 CSV，先確認檔案建立時間或修改時間至少一個為台北時區今天。
4. 從產品資料 CSV 同資料夾讀取 `產品比對身份關鍵詞.csv`，用其中的 `版本詞`、`角色機體` 強化重複辨認。
5. 若試算表的 `產品代號` 欄已有值，必須先用今天建立的產品資料 CSV 驗證該代號是否存在：
   - 代號存在：狀態寫為 `已建檔`，可沿用該產品代號。
   - 代號不存在：不得視為已建檔，改用品名判斷狀態為 `確認為新品` 或 `有類似產品`。
   - 代號空白：視為使用者保留手動填入空間，不影響比對狀態，改用品名判斷狀態為 `確認為新品` 或 `有類似產品`。
6. 執行 `scripts/match-existing-products.py`，由 RapidFuzz 先計算本地相似度，再套用版本詞、角色機體等身份規則調整分數；RapidFuzz 不可用時才退回腳本內建相似度。預設候選門檻為 `0.6`，避免把只有平台相同但商品不同的低可信項目丟給人工確認；特殊 OCR 品質很差時才手動調低。輸出檢查用 xlsx 到輸入試算表同一個目錄，檔名為原檔名加 `_產品比對檢查.xlsx`，並在原欄位右側寫入：`比對狀態`、`已建檔代號`、`已建檔品名`、`相似候選`。不要輸出 `產品代號檢查` 或 `比對說明` 欄。
7. 若 `比對狀態` 為 `已建檔` 且原 `產品代號` 空白，自動把 `已建檔代號` 回填到 `產品代號`，並將儲存格格式設為文字以保留前導 0。
8. 輸出 xlsx 的儲存格欄寬必須自動配合內容調整，`相似候選` 必須每個候選一行並自動換行，避免結果被遮住。
9. 逐筆覆核輸出 xlsx 的比對結果：
   - `已建檔`：可直接沿用既有產品代號，後續採購單要帶入。
   - `有類似產品`：只列候選，需使用者確認是否同商品。
   - `確認為新品`：沒有合理候選，需建立新商品。
10. 若試算表已填產品代號且該代號存在，但進貨品名與該代號正式品名相似度過低，不改變 `已建檔` 狀態、不新增欄位；在 `比對狀態` 與 `已建檔品名` 儲存格加淡黃色底色與註解，提醒人工覆核是否填錯代號。
11. 只有存在 `有類似產品` 或 `確認為新品` 時，建立可複製的建議名稱 `.txt`。每行格式為 `()建議名稱`。若全部商品都是 `已建檔`，不得留下舊的建議名稱 `.txt`。
12. 全部商品都是 `已建檔` 時，不建立 `.txt`，直接交給 `$build-inventory-import-files` 產生採購匯入檔。`有類似產品` 只代表需要人工判斷，不可自動當新品；使用者可回覆新產品代號與正式名稱，或指定沿用相似候選的既有代號。

## 腳本

```powershell
python .\scripts\match-existing-products.py `
  --input-xlsx "C:\path\to\建檔進貨用\進貨圖片轉試算表\進貨單.xlsx" `
  --csv "C:\path\to\參考資料\產品資料輸出.CSV"
```

若需要覆寫輸出位置才使用 `--output-xlsx`；日常流程不要指定，讓檢查檔留在輸入檔同目錄。
