# 產品比對與命名規則

## 建議名稱

1. 保留原始品名，另建 `suggestedName` 或正式名稱。
2. 若判斷為盒玩，在名稱前加 `(盲盒)`，並在最後加 `@數字` 表示中盒數。
3. 移除不需要的附加文字，例如 `(全6種)`、`(再販)`、`(單)`。
4. 使用 `-` 分隔主要名稱區塊。
5. 建議名稱文字檔只在名稱前加空白 `()`，不要自行填入括號內分類。
6. 完整進貨流程使用 `--no-suggestion-txt`，不建立建議名稱文字檔；確認名稱改由覆核技能在對話輸出。單獨執行產品比對時仍保留文字檔功能。

## 既有商品比對

1. 比對前確認產品資料 CSV 的檔案建立時間或修改時間至少一個是台北時區今天；檔名不需要包含日期。
2. 正規化圖片品名、推論名稱與 CSV 產品名稱：忽略分類括號、空白、常見分隔符、全半形差異，以及不影響身份的附加文字。
3. 候選分數預設使用本地 RapidFuzz 計算，不使用 AI 逐筆判定是否為同商品；RapidFuzz 不可用時才退回腳本內建相似度。
4. 預設候選門檻為 `0.6`；低於門檻的項目不列入 `相似候選`，避免只因平台或少數通用字相同就增加人工覆核負擔。
5. 名稱核心、作品、系列與規格足以確認為同一商品時，標記 `exact`。
6. 只有作品、系列或關鍵字接近時，標記 `similar`，不得自行沿用代號。
7. 沒有合理候選時，標記 `new`。
8. `exact` 不列入建議名稱 `.txt`，也不寫入建檔用 `.xls`。
9. 比對結果寫回檢查用 xlsx，不輸出 JSON。
10. 檢查用 xlsx 必須放在輸入試算表同一個目錄，預設檔名為原檔名加 `_產品比對檢查.xlsx`。
11. 輸入檔名若已包含 `_產品比對檢查`，代表是已產生的檢查檔，必須停止並要求改用原始進貨試算表。
12. 輸出欄位只新增 `比對狀態`、`已建檔代號`、`已建檔品名`、`相似候選`；不得新增 `比對說明` 或獨立的 `產品代號檢查` 欄。
13. `比對狀態` 只判斷商品是否已建檔或可能重複，使用中文顯示：`確認為新品`、`有類似產品`、`已建檔`；不得合併產品代號檢查文字。
14. 產品代號空白代表保留給使用者手動填入，不要因此新增 `建議建檔名稱` 欄。
15. 試算表已填產品代號且代號存在時，仍沿用該代號；若進貨品名與該代號正式品名相似度低於 `0.35`，不得新增欄位或改成新品，只在 `比對狀態` 與 `已建檔品名` 加淡黃色底色與註解，提醒人工覆核可能填錯代號。
16. 輸出 xlsx 必須自動調整欄寬，`相似候選` 必須自動換行。

## 重複辨認規則

1. 商品是否重複以身份關鍵資訊優先，依序檢查：作品、系列、版本詞、角色或機體、規格型態。
2. 版本詞是商品身份的一部分，例如 `奇蹟版`、`超越版`、`群星版`、`菁英版`、`駕駛服`。版本詞不同時，即使品牌與作品相同，也要降低相似分數，避免誤判為同商品。
3. 只有品牌或作品相同，但版本詞、角色或規格不同時，最多列為低可信候選；分數低於門檻時應判為 `確認為新品`。
4. 角色或機體名稱一致、作品一致、版本詞一致時，才可判為高可信相似或 exact。
5. 角色或機體名稱必須納入權重：一致時提高相似可信度，不一致時大幅降低；進貨品名有角色但候選沒有角色時，也要降低分數。
6. `相似候選` 每個候選一行，格式為 `產品代號 產品名稱 (分數)`，不要用分號串在同一行。

## 身份關鍵詞清單

1. 使用 `參考資料/產品比對身份關鍵詞.csv` 維護身份詞，不要把詞寫死在腳本。
2. CSV 欄位為 `類型`、`關鍵詞`、`啟用`、`備註`。
3. `類型` 目前支援 `版本詞`、`角色機體`。
4. `啟用` 填 `Y` 或空白代表使用；填 `N`、`NO`、`0`、`FALSE` 代表停用。

## 大類

使用最新產品資料 CSV 推論，優先比對同型態、同作品、同系列商品。一般 `(盲盒)` 常見大類為 `34`，但不得只因為是盲盒就強制填 `34`。
