# 進貨單覆核調整

## 目標

覆核 `$match-product-catalog` 產生的產品比對檢查 `.xlsx`。本技能先依產品資料 CSV 與大類清單產生「修正後品名、預估括號分類、大類」，讓使用者先確認命名並填入產品代號；只有在正式輸出 `[調整].xlsx` 時，才要求每筆商品都有合法 `產品代號`。

## 本地引擎

- 使用 Python/openpyxl 讀寫 `_產品比對檢查.xlsx`、待確認內容與 `[調整].xlsx`。
- 使用 Python `csv` 讀取 `產品資料輸出*.CSV`、`品牌括號命名規則.csv` 與 `大類清單.csv`。
- 不使用 Excel COM 建立或修改中間 `.xlsx`；Excel COM 只由 `$build-inventory-import-files` 在正式 `.xls` 成品輸出時使用。

## 工作流程

1. 接收使用者提供的 `_產品比對檢查.xlsx`，以及使用者指定或工作區內最新的 `參考資料/產品資料輸出*.CSV`。未指定 `--csv` 時，腳本會自動尋找工作區 `參考資料/產品資料輸出*.CSV` 或 `產品資料輸出.CSV`。
2. 讀取試算表表頭列，必要欄位為 `產品代號`、`品名`。若缺欄，立即停止並列出缺少欄位；若沒有 `大類` 欄，先自動新增。
   - 讀取商品列時，若 `產品代號` 或 `品名` 欄出現 `總價格`、`總價`、`總計`、`合計`、`小計`，該列視為摘要列，不得檢查產品代號或列入調整輸出。
3. 先參考 `產品資料輸出*.CSV` 內的 `2.產品名稱`、本技能 `references/大類清單.csv` 與工作區 `參考資料/品牌括號命名規則.csv`，調整每筆 `品名` 並推論 `大類`。此階段允許 `產品代號` 空白。
   - `比對狀態` 為 `已建檔` 的商品，不重新推論命名；若有 `已建檔品名`，直接使用該既有正式品名。
   - 使用者已填入六位產品代號，且品名已是括號開頭的正式名稱時，保留使用者貼上的品名，不再重新推論命名。
   - 格式為 `(分類)角色或主題名稱-作品名稱廠商系列名稱`。
   - `()` 內放品項分類，例如 `(景品)`、`(模型)`、`(盲盒)`；無法可靠推論時保留空括號 `()`，不要硬猜。
   - 原產品名稱含 `參考資料/品牌括號命名規則.csv` 內啟用的關鍵字時，將該文字移到前方 `()` 內，並從後方名稱移除。
   - `-` 前放角色、機體、主題或商品主體。
   - `-` 後放作品名稱，再接廠商賦予的系列、版本、規格名稱。
   - 參考格式例：`(景品)艾德華·紐蓋特-航海王KING OF ARTIST`。
4. 依產品屬性與參考 CSV 推論 `大類`：
   - `比對狀態` 為 `已建檔` 的商品不強制補大類，因為不會寫入建檔用檔案。
   - 優先讀取 `references/大類清單.csv`，用 `品名` 前方括號分類與關鍵字推論大類。例如 `(景品)` 對應 `39`。
   - 再找同作品、同型態、同系列的既有商品；若產品資料 CSV 沒有明確大類欄，仍以 `大類清單.csv` 為主。
   - 無法可靠推論時保留原本大類；若原本也空白，列入確認清單，並在對話中列出大類清單供使用者參考。
5. 輸出確認清單前先排除一番賞與抽賞商品：
   - 只有原始品名、正式品名、已建檔品名或使用者已填入的建議建檔名稱本身出現 `一番賞`、`抽賞`、`Ichiban Kuji` 時，該商品才不列入待填代號清單，不輸出到 `[調整].xlsx`，也不交給建檔或採購匯入。
   - 不得因 `相似候選` 內含一番賞或抽賞商品就排除該列；相似候選只供人工比對，不能作為排除依據。
   - 原始品名或備註出現 `遮蔽`、`已遮蔽`、`人工確認重複`、`重複品項` 等標記時，也視為排除項目。
   - 排除時在對話中列出排除品名，方便使用者確認本次不處理。
6. 先在對話中輸出待填代號確認清單，欄位只顯示 `產品代號`、`產品名稱`、`大類`。
   - `產品代號` 可先空白，讓使用者依修正後名稱與大類填入代號。
   - `大類` 顯示時在代號後方加上中文名稱，例如 `38 公仔/吊卡/PVC`，方便人工檢查。
   - 另提供可複製文字區時，若產品代號尚未填入，不要在產品名稱前放空白欄、tab 或空格；直接輸出 `產品名稱<TAB>大類代號 中文名稱`，方便使用者複製後自行在最前方補產品代號。
   - 使用者要修改時，可直接貼上對應的代號、名稱或大類；依使用者貼上的內容更新後，再重新輸出確認清單。
   - 使用者貼回的正式名稱或大類只要與原始品名/原推論大類不同，就分析這次修正中的括號分類、名稱位置、產品型態與大類，更新到工作區 `參考資料/品牌括號命名規則.csv`；不要再建立或更新 `進貨命名修正紀錄.csv`。
   - `品牌括號命名規則.csv` 是命名推論的優先規則表；先套用此表，再使用一般命名推論，最後才輸出給使用者修正。
   - 未取得產品代號與使用者確認前，不要產生正式調整檔。
   - 使用者回覆後，將有效資料整理為 UTF-8 TSV 暫存檔，每行 `產品代號<TAB>產品名稱<TAB>大類`。執行本技能時用 `--answers-tsv` 直接套用到記憶體中的原 `_產品比對檢查.xlsx`，不要建立 `_已套用.xlsx`。
7. 正式輸出 `[調整].xlsx` 前，才檢查每筆商品列的 `產品代號`：
   - 必須有內容。
   - 必須是六位數字文字，前面可以是 `0`。
   - 若 `比對狀態` 為 `已建檔` 且 `產品代號` 空白，但 `已建檔代號` 有六位數代號，先自動回填到 `產品代號`。
   - 任何一列缺漏或格式錯誤時，停止後列出列號、品名、目前值，請使用者填入或修正；不要輸出 `[調整]` 檔。
   - 任何非 `已建檔` 商品的大類仍空白就停止，請使用者補大類，並列出大類清單供參考。
8. 執行 `scripts/review-invoice-product-check.py --confirmed` 產生調整檔，預設輸出到輸入檔同一資料夾，檔名為原檔名加 `[調整].xlsx`。輸出檔必須保留 `比對狀態`，供 `$build-inventory-import-files` 判斷哪些列只進採購、哪些列需建檔；其他比對暫存欄位 `建檔代號`、`已建檔代號`、`已建檔品名`、`相似候選` 不保留。
9. 重新開啟輸出檔驗證：
   - `產品代號` 仍為六位數字文字，不可被 Excel 轉成數字而吃掉前導 0。
   - `品名` 已改為調整後名稱。
   - `大類` 有值。
   - 參考 `$match-product-catalog` 的可讀性規則，欄寬自動配合內容、文字可換行、表頭清楚可讀。

## 腳本

```powershell
python "C:\Users\user\.codex\skills\review-invoice-product-check\scripts\review-invoice-product-check.py" `
  --input-xlsx "C:\path\to\進貨單_產品比對檢查.xlsx"
```

若工作區內有多份產品資料 CSV，或要指定特定檔案，再加入 `--csv "C:\path\to\參考資料\產品資料輸出.CSV"`。

品牌括號規則預設讀取工作區 `參考資料/品牌括號命名規則.csv`；若要指定其他清單，加入 `--brand-rules "C:\path\to\品牌括號命名規則.csv"`。CSV 欄位固定為：`關鍵字`、`括號名稱`、`大類`、`位置規則`、`命名規則`、`範例原始名稱`、`範例調整後名稱`、`啟用`、`說明`。

大類清單預設讀取本技能的 `references/大類清單.csv`；若要指定其他清單，加入 `--category-rules "C:\path\to\大類清單.csv"`。CSV 欄位固定為：`大類代號`、`大類名稱`、`括號分類`、`關鍵字`、`啟用`、`備註`。

預設會先輸出「待填代號確認清單」，即使產品代號空白也會先產生修正後名稱、預估括號分類與大類；不建立試算表。使用者補齊產品代號並確認正確後才執行：

```powershell
python "C:\Users\user\.codex\skills\review-invoice-product-check\scripts\review-invoice-product-check.py" `
  --input-xlsx "C:\path\to\進貨單_產品比對檢查.xlsx" `
  --answers-tsv "C:\path\to\覆核回覆.tsv" `
  --confirmed
```

`--answers-tsv` 可包含全部商品，或只包含尚未標記為 `已建檔` 的待確認商品；筆數必須和其中一種相符。第三欄可填 `39` 或 `39 景品`，腳本只寫入前方大類代號。代號若已存在於產品資料 CSV，該列改為 `已建檔`；不存在則改為 `確認為新品`。輸出直接由原比對檢查檔建立 `[調整].xlsx`。

若要只檢查產品代號、不做名稱與大類推論、不輸出調整檔：

```powershell
python "C:\Users\user\.codex\skills\review-invoice-product-check\scripts\review-invoice-product-check.py" `
  --input-xlsx "C:\path\to\進貨單_產品比對檢查.xlsx" `
  --check-only
```

詳細命名與推論規則見 [references/adjustment-rules.md](references/adjustment-rules.md)。
