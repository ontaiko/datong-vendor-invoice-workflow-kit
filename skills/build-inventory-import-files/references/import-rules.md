# 建檔與採購匯入規則

## 建檔用

只寫入尚未建檔的新商品：

- 若輸入 `.xlsx` 有 `比對狀態` 欄，`已建檔` 不寫入建檔用，其他列寫入建檔用。
- 若輸入 `.xlsx` 沒有 `比對狀態` 欄，視為已由 `$review-invoice-product-check` 輸出的調整後建檔資料，全部列入建檔用。
- `產品代號`：確認後代號
- 產品代號由 `$review-invoice-product-check` 與使用者事先確認；本技能只依輸入 `.xlsx` 現有內容寫入，不檢查六位數格式、不補代號。
- 寫入 `產品代號` 前，先將建檔用試算表第 1 欄設為文字格式，避免純數字代號前面的 `0` 被 Excel 自動刪除。
- `產品名稱`：確認後名稱
- `銷售單價1`、`銷售單價2`：`0`
- `建議售價`：單據沒有時 `0`
- `單位`：`PCS`
- `成本`：進價或含稅修正後成本
- `CO128`：同產品代號
- `大類`：依輸入 `.xlsx` 現有內容寫入；本技能不推論、不補值。
- 範本若包含名為 `分類` 的工作表，輸出成品前必須刪除，只保留正式建檔匯入工作表。
- 同一次流程若有多個廠商，建檔用只產生一份合併檔，將所有廠商的新商品彙整在同一份建檔用；不要依廠商分開產生多份建檔用。

## 採購單匯入

- `採購日期`：未指定時使用台北時區今天，格式民國 `yyy.MM.dd`
- `廠商代號`：優先參考 `參考資料/廠商代號.xlsx`，完全依廠商代號表格式輸入；未手動指定時，腳本自動用廠商簡稱查表。
- `萬榮`、`萬榮國際` 的採購單廠商代號固定使用 `38`，即使廠商代號表另有包含萬榮字樣的其他候選。
- `產品代號`、`數量`、`單價`：填確認後資料
- 採購單匯入只接受輸入 `.xlsx` 的 `產品代號` 欄，不從 JSON 或其他暫存檔補代號。
- 寫入 `產品代號` 前，先將採購單匯入的產品代號欄設為文字格式，避免純數字代號前面的 `0` 被 Excel 自動刪除。
- `單位`：`pcs`
- `備註1`：`阿榮代打`
- 其他未標註欄位留白
- 多廠商時採購單匯入仍依廠商分開產生，因為每份採購單需要對應自己的廠商代號。

## 成本與空白列

- 建立任何目標 `.xls` 前，必須先停下來請使用者確認已檢查並調整完進貨單資料。
- 先核對 `數量 × 進價` 是否符合列印金額。
- 若不符合但 `數量 × 進價 × 1.05` 符合，成本改為 `進價 × 1.05`。
- 廠商簡稱或來源廠商命中 `南波`、`南波丸` 時，成本固定改為 `進價 × 1.05`；這是廠商固定規則，不需要等金額不符合才套用。
- 兩種算法都無法吻合時停止，請使用者確認。
- 產生 `.xls` 後刪除最後一筆商品以下的多餘空白列。
- 輸出檔不得覆蓋既有檔案；檔名使用 `[廠商簡稱][建檔用or採購單用][日期]-[流水號].xls`。
- 同一次產生建檔用與採購單用時，兩個檔案使用同一個流水號；任一檔案已存在時改用下一組流水號。
- 多廠商合併建檔用檔名的廠商簡稱使用該次流程的共同簡稱；若沒有共同簡稱，使用 `多廠商`。

## 中間檔清理

- 本技能只負責產生成品與驗證成品，不主動刪除來源 `.xlsx` 或其他中間檔。
- 若流程由 `$convert-vendor-invoice-image` 管理，正式成品驗證完成後，由總管技能依本次流程檔名前綴刪除原始明細、比對檢查、建議名稱、已套用、調整與含稅成本等中間檔。
- 若正式成品產生失敗或無法驗證，保留所有中間檔供排查。
