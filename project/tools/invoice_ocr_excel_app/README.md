# 進貨單圖片轉 Excel

本工具是本機穩定版小工具，雙擊 `進貨單圖片轉Excel.exe` 後可以選擇一張進貨單圖片，指定輸出資料夾，依序完成 OCR 原文確認、已建檔勾選、名稱調整、建檔用與採購單匯入檔輸出。

## 使用方式

1. 開啟 `進貨單圖片轉Excel.exe`。
2. 選擇單張進貨單圖片，支援 `.jpg`、`.jpeg`、`.png`、`.webp`。
3. 選擇輸出資料夾，預設為 `建檔進貨用\進貨圖片轉試算表`。
4. 按 `開始 OCR + 產品比對`。
5. 在 `OCR 原始文字確認頁` 修改 OCR 品名、數量、進價、金額。
6. 若某列已經是已建檔商品，勾選 `已建檔`，並確認六位產品代號與已建檔品名。
7. 按 `確認文字並開始調整`。
8. 在 `名稱調整頁` 確認新品或類似品的產品代號、產品名稱、大類、數量、成本、金額。
9. 若需要拆分品項，選取來源列後按 `拆分/新增品項`，再調整拆分後數量與金額。
10. 按 `確認名稱並產生調整檔`。
11. 確認資料正確後，按 `產生建檔/採購檔`。
12. 成品產生後，可依提示清理中間檔。

## 流程範圍

- OCR 後先在程式內確認原始文字，不強制使用者開啟原始 Excel。
- 自動接產品比對，輸出 `_產品比對檢查.xlsx`。
- 已建檔商品可在 OCR 原始確認頁勾選，跳過名稱調整頁，直接進採購單輸出資料。
- 在程式內只覆核需要建檔或調整的產品代號、正式品名與大類。
- 大類在介面中顯示為 `代號 中文名稱`，例如 `38 公仔/吊卡/PVC`。
- 名稱調整頁可拆分品項，拆分後會檢查來源列數量與金額合計。
- 產生 `[調整].xlsx`。
- 經使用者確認後產生建檔用 `.xls` 與採購單匯入 `.xls`。
- 成品產生後才可清理中間檔。

## 內建參考資料

參考資料放在：

```text
tools\invoice_ocr_excel_app\reference_data
```

目前包含：

- `產品資料輸出.CSV`
- `產品比對身份關鍵詞.csv`
- `品牌括號命名規則.csv`
- `大類清單.csv`
- `廠商代號.xlsx`
- `建檔用.xls`
- `採購單匯入範例.xls`

產品比對會使用 `產品資料輸出.CSV` 與 `產品比對身份關鍵詞.csv`。名稱調整會使用 `品牌括號命名規則.csv` 與 `大類清單.csv`。正式輸出會使用 `廠商代號.xlsx`、`建檔用.xls`、`採購單匯入範例.xls`。

產品資料 CSV 需要是當天建立或修改的版本。程式會在 OCR 前檢查；若隔天使用，請先把新的 `產品資料輸出.CSV` 放進 `reference_data`。

## 停止點

- 產品資料不是今天版本時，會在 OCR 前停止。
- OCR 有疑點時仍會保留疑點摘要，請先在原始文字確認頁檢查。
- 新品或類似品必須在表格中確認產品代號、正式品名與大類。
- 已建檔勾選必須有六位產品代號與已建檔品名。
- 拆分品項後，來源列的數量與金額合計不一致時會停止。
- 正式輸出前會跳出確認，確認後才會產生 `.xls`。
- 正式成品存在後才允許清理中間檔。

## OCR 環境

程式固定使用全系統 Python 與 PaddleOCR：

```text
C:\Users\user\AppData\Local\Programs\Python\Python312\python.exe
```

不再依賴專案內 `.venv-paddleocr`。

## 重新打包

在 PowerShell 執行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\tools\invoice_ocr_excel_app\build_exe.ps1"
```

打包完成後會產生：

```text
tools\invoice_ocr_excel_app\進貨單圖片轉Excel.exe
```
