# 大統進貨流程記憶種子

這份內容給同事的 Codex 作為專案操作記憶，不是聊天紀錄。

## 使用者情境

- 使用者是電玩店工作者，常處理廠商進貨單、商品建檔、產品比對、採購匯入與庫存行政。
- 回覆預設使用繁體中文。
- 目標是直接產生可用檔案與可複製清單，少講理論。

## 進貨流程

- 圖片像廠商進貨單、銷貨憑單、送貨單時，啟動 `convert-vendor-invoice-image`。
- 總管技能只負責協調，不覆寫子技能規則。
- 子技能順序：
  1. `extract-vendor-invoice-image`
  2. `match-product-catalog`
  3. `review-invoice-product-check`
  4. `build-inventory-import-files`

## 停止點

- OCR 有疑點時先回報，不直接進產品比對。
- 產品資料 CSV 不是今天版本時先停止。
- 新品、相似產品、缺產品代號或缺大類時，先給使用者確認清單。
- 建立正式建檔用與採購匯入檔前，必須確認使用者已檢查 `[調整].xlsx`。
- 一番賞、抽賞、遮蔽且已確認重複的品項排除，不要求補代號。

## 檔案與資料

- 專案預設位置：`%USERPROFILE%\Documents\大統工作助手`
- 參考資料放在：`參考資料`
- OCR 原始轉表預設放在：`建檔進貨用\進貨圖片轉試算表`
- 本地 OCR 環境放在：`.venv-paddleocr`

## 回報格式

正式輸出後固定摘要：

- 商品筆數
- 新品筆數
- 已建檔筆數
- 採購筆數
- 廠商代號
- 含稅調整
- 排除項目
- 特殊處理
